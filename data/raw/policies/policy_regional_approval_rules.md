# Regional Approval Rules

## Scope
This document describes regional differences in approval workflow behavior for quote-to-cash support teams.

## Germany and EU rules
Germany and EU teams use the common approval thresholds, but workflow routing may include region-specific queues and release-controlled restrictions.
When a release note changes routing behavior for Germany or EU accounts, support should use that release note together with current policy documents.

## Operational routing
Cases involving approval failures in Germany should be routed to the Germany support queue first if a region-specific incident is active.
Support may not rely on legacy manual override guidance unless the current policy explicitly allows it.

## Document precedence
When sources conflict, use the following order of precedence:
1. current policy,
2. current release note,
3. troubleshooting guide,
4. support KB,
5. meeting notes.

## Why this matters
This precedence rule is intended to reduce confusion after regional workflow changes.
